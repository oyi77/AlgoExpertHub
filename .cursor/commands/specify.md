# /specify Command

Create a comprehensive feature specification document.

## Usage

/specify [feature-name]

## Purpose

Generate a detailed specification document that captures all requirements, user stories, edge cases, and success criteria for a feature.

## Workflow

1. Analyze existing feature brief or research (if available)
2. Create comprehensive spec document
3. Include: user stories, requirements, edge cases, error scenarios, success criteria

## Prerequisites

- Feature brief (`specs/active/[feature-name]/feature-brief.md`) OR
- Research document (`specs/active/[feature-name]/research.md`)

## Example

/specify payment-system

## Output

- `specs/active/[feature-name]/spec.md` - Complete specification document
