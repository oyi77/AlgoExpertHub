# /plan Command

Create a technical architecture and implementation plan.

## Usage

/plan [feature-name]

## Purpose

Generate a detailed technical plan that includes architecture decisions, technology choices, implementation approach, and technical considerations.

## Workflow

1. Analyze specification document
2. Research technical approaches (if not already done)
3. Create architecture plan with technology choices and justification
4. Include: architecture, tech stack, security, performance, testing strategy

## Prerequisites

- Specification document (`specs/active/[feature-name]/spec.md`)
- Optional: Research document (`specs/active/[feature-name]/research.md`)

## Example

/plan payment-system

## Output

- `specs/active/[feature-name]/plan.md` - Technical architecture and implementation plan
