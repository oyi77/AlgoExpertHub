# /evolve Command

Update and evolve specifications as development progresses.

## Usage

/evolve [feature-name] [changes-description]

## Purpose

Keep specifications aligned with actual implementation. Update specs when requirements change or new discoveries are made during development.

## Workflow

1. Analyze current implementation vs specifications
2. Update relevant SDD documents (spec, plan, tasks)
3. Document changes and rationale
4. Maintain version history

## Example

/evolve payment-system Added support for Apple Pay based on user feedback

## Output

- Updated specification documents
- Change log entries
