# /research Command

Research and document best practices, technologies, and patterns for a feature.

## Usage

/research [feature-name] [research-topics]

## Purpose

Conduct research on technologies, patterns, and best practices before creating specifications and plans.

## Workflow

1. Research the specified topics and technologies
2. Document findings, comparisons, and recommendations
3. Create research document with actionable insights

## Example

/research payment-system Payment gateways, security best practices, PCI compliance

## Output

- `specs/active/[feature-name]/research.md` - Research findings document
