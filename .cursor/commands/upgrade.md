# /upgrade Command

Upgrade a feature brief to full SDD 2.0 workflow.

## Usage

/upgrade [feature-name]

## Purpose

Convert a lightweight feature brief into a full specification-driven development workflow when complexity increases.

## Workflow

1. Analyze existing feature brief
2. Generate full specification document
3. Create research and plan documents
4. Break down into tasks

## Prerequisites

- Feature brief (`specs/active/[feature-name]/feature-brief.md`)

## Example

/upgrade user-authentication

## Output

- Full specification (`spec.md`)
- Research document (if needed)
- Plan document (`plan.md`)
- Tasks document (`tasks.md`)
